# 文法与任务书对照说明

## 1. 阶段二与阶段三下标规则

| 来源 | 产生式 | 本项目实现 |
|------|--------|------------|
| 实验阶段二（任务书） | `Loc → loc[num] \| id` | 合并为综合实验后采用阶段三规则 |
| 实验阶段三（任务书） | `Loc → loc[bool] \| id` | `loc → loc[boolExpr] \| id` |

**说明：**

- 综合实验以**阶段三**为准，下标为**布尔表达式**（`boolExpr`），可写变量下标 `arr[i]` 或常量下标 `arr[0]`（`0` 属于 `factor → num`，进入 `expr` 再进入 `boolExpr`）。
- 非终结符命名为 `boolExpr`，对应任务书中的 `bool`，仅为避免与关键字 `bool` 混淆。
- 常量下标在词法层为 `num`，在语法层通过 `expr`/`boolExpr` 推导，与“仅允许 `num` 终结符”在表达能力上等价，且符合阶段三“布尔表达式下标”的扩展要求。

## 2. 数组声明

任务书：`type → type[num] | basic`，`decl → type id;`

本项目文法与之一致，源程序写法为：

```text
int[10] arr;
```

而非 C 风格的 `int arr[10];`。

## 3. 合并编译器验收口径

答辩与实验报告以**三阶段合并程序** `CompilerApplication` 为准：词法 → LR(1) 语法分析（含分析表与栈）→ 语法制导四元式 + 语义检查。阶段二单独提交的口径见上表“阶段二→阶段三合并”说明。
